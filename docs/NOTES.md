# Заметки по реализации

Первый MVP сознательно использует системный `ssh` как дочерний процесс. Это временное решение: оно уже позволяет контролировать процесс, не запускать `ssh -f`, проверять реальную работоспособность SOCKS и перезапускать зависший туннель.

Целевой вариант — встроенный SSH backend на Go. Тогда программа сможет полностью управлять TCP-соединениями без внешнего процесса `ssh`.

## Failover

Одиночный профиль и группа работают по-разному.

- `tunnelctl connect yandex --watch` переподключает тот же профиль `yandex`.
- `tunnelctl connect auto --watch` запускает профили группы по очереди. Если текущий VPS завершился, не открыл SOCKS-порт или провалил health-check, управление возвращается супервизору группы, и он переходит к следующему профилю.
- Если все профили группы завершились ошибкой, начинается новый круг с экспоненциальной задержкой из настроек `defaults.reconnect`.

Все пользовательские сообщения, комментарии и логи в проекте должны быть на русском языке.
