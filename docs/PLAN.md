# План развития tunnelctl

## Реализовано

- стартовые скрипты `start.sh`, `start.cmd`, `start.ps1`;
- конфигурация в стандартном месте ОС;
- импорт `ssh -D` команд из истории;
- генерация алиасов;
- управляемый запуск системного `ssh` без `-f`;
- SOCKS health-check и перезапуск;
- failover-группы;
- локальный управляющий канал текущего пользователя;
- команды `status`, `stop`, `switch <profile-or-alias>`, `switch next`;
- безопасный state-файл без SSH-секретов;
- автозапуск через единый CLI;
- Linux user/system `systemd`;
- Планировщик заданий Windows;
- Termux:Boot;
- предварительный просмотр `autostart print` и `--dry-run`;
- защита чужих объектов и идемпотентная установка.

## Следующие задачи

1. Встроенный SSH backend на Go без внешнего процесса `ssh`.
2. Более точная диагностика владельца занятого SOCKS-порта.
3. Release-архивы с готовыми бинарниками и подписанными контрольными суммами.
4. Расширенные интеграционные тесты на реальных Ubuntu, Debian, Windows и Termux.
5. Проверка и документирование `systemd` lingering для пользовательского автозапуска до входа.
6. Дополнительная защита Windows named pipe с явно сформированным ACL текущего SID.
7. Команда просмотра истории переключений и причин failover.
8. Возможность нескольких независимых экземпляров с разными SOCKS-портами и именованными runtime-контекстами.
9. Команда `profile test` или режим `doctor --connect` с проверкой SSH, SOCKS и health URL.
10. Проверка импортированных профилей до автоматического включения в failover-группу.
11. Явная обработка конфликтов одного endpoint с разными ключами при импорте истории.
12. Команды добавления, изменения и удаления профилей и состава групп без ручного редактирования JSON.

Практические наблюдения и критерии приёмки: [`ANDROID_TERMUX_FINDINGS.md`](ANDROID_TERMUX_FINDINGS.md).

## Поддержка устройств Apple

- macOS: установка и управление пользовательским `LaunchAgent` через `launchd`;
- iOS/iPadOS: исследование допустимых механизмов постоянного SSH/SOCKS-туннеля с учётом ограничений фонового выполнения;
- исследование Shortcuts и автоматизаций;
- исследование VPN Network Extension и других разрешённых Apple механизмов;
- отдельная проверка жизненного цикла, энергосбережения и ограничений App Store;
- обычный постоянно работающий фоновый процесс на iOS/iPadOS может быть ограничен системой;
- поддержка iPhone и iPad не считается реализованной до отдельного исследования и практической проверки.
