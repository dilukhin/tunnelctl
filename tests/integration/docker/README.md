# Docker-интеграция

Сценарий поднимает два изолированных SSH-сервера и HTTP-ресурс, который не публикуется на хосте. `tunnelctl` запускается на хосте теста и должен получить ресурс через реальный SOCKS5-туннель.

Проверяются:

- начальное подключение к первому профилю группы;
- реальный HTTP-запрос с удалённым разрешением имени через SOCKS5;
- автоматический failover после остановки первого SSH-сервера;
- `switch next` после восстановления первого сервера;
- явный `switch` на второй профиль;
- `stop`, завершение дочернего `ssh`, удаление state-файла и управляющего сокета, освобождение порта.

Тестовый SSH-ключ генерируется во временном каталоге при каждом запуске и в репозиторий не записывается.

Локальный запуск из корня репозитория:

```bash
bash tests/integration/docker/run.sh
```

Необходимы Go, Docker с Compose v2, OpenSSH client и `curl` с поддержкой SOCKS5.
